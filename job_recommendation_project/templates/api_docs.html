REST API Documentation

Full JSON API for NexaJobs. Use JWT tokens for authenticated endpoints.

BASE URL http://127.0.0.1:8000/api/v1
🔐 Authentication
POST /auth/register/ Create a new account
FieldTypeRequiredDescription
usernamestring✓Unique username
emailstring✓User email
first_namestring✓First name
last_namestring✓Last name
passwordstring✓Min 8 characters
password2string✓Must match password
// Example response { "user": { "id": 1, "username": "johndoe", "email": "john@example.com" }, "access": "eyJ0eXAiOiJKV1QiLC...", "refresh": "eyJ0eXAiOiJKV1QiLC...", "message": "Account created successfully!" }
POST /auth/login/ Get JWT tokens
FieldTypeRequired
usernamestring✓
passwordstring✓
// Returns access + refresh tokens { "user": {...}, "access": "...", "refresh": "..." }
POST /auth/refresh/ Refresh access token
// Body: { "refresh": "your_refresh_token" } // Returns: { "access": "new_access_token" }

🤖 AI Recommendations
GET /recommendations/ Get AI-ranked jobs for current user 🔑 Auth required

Returns jobs ranked by TF-IDF + Cosine Similarity + Collaborative Filtering.

// Add header: Authorization: Bearer <access_token> { "count": 20, "results": [{ "id": 3, "title": "...", "ai_score": 0.87, "match_percent": 75, ... }] }
👤 Profile & Feedback
GET /profile/ Get current user's profile 🔑 Auth required
// PATCH /profile/ to update — send only fields you want to change { "skills": "Python, Django", "preferred_roles": "Backend Developer", ... }
GET /feedback/ Get user's liked/applied/saved jobs 🔑 Auth required
Query ParamValues
actionlike, save, apply, skip
// POST /feedback/ to submit feedback { "job": 3, "action": "like" }
GET /stats/ Dashboard stats for current user 🔑 Auth required
{ "total_jobs": 12, "applied": 3, "liked": 5, "saved": 2, "profile_complete": true }
📖 How to Use JWT Auth
// 1. Login to get tokens POST /api/v1/auth/login/ { "username": "johndoe", "password": "yourpassword" } // 2. Use access token in every authenticated request GET /api/v1/recommendations/ Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhbGc... // 3. When access token expires (1 day), refresh it POST /api/v1/auth/refresh/ { "refresh": "your_refresh_token" }
